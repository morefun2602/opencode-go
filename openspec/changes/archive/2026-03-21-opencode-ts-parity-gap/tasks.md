# 任务清单：与上游 TypeScript OpenCode 功能对齐

实现顺序按依赖排列；勾选表示完成。详细行为以 `specs/**/*.md` 与 `design.md` 为准。

## 1. 仓库结构与依赖边界

- [x] 1.1 按 `go-codebase-layout` 增量规格新增 `internal/mcp`、`internal/tool`（或 `internal/builtin`）、`internal/skill`、`internal/plugin`、`internal/acp` 等占位包，并确保 `go build ./...` 无循环依赖
- [x] 1.2 为横切包补充文档说明「禁止 import `cmd` / HTTP handler 包」的依赖方向，并在 CI 或脚本中可选运行 `go vet`/模块图检查

## 2. 持久化与上游语义映射

- [x] 2.1 编写「上游逻辑实体 → 本地表/列」映射文档（会话、消息、turn、附件元数据等），并与 `persistence` delta 中的不变量交叉引用
- [x] 2.2 实现或调整迁移：支持消息版本/`schema` 字段（与 `agent-runtime` delta 一致），并为大版本不兼容数据提供启动时检测与明确错误信息
- [x] 2.3 为会话列表与消息分页查询添加索引或等价查询计划，并补充可重复执行的性能或契约测试

## 3. LLM 提供商与工具路由骨架

- [x] 3.1 实现可扩展的 LLM 提供商注册表与配置选择（`llm-and-tools` delta），确保 `llm` 不直接依赖具体工具实现
- [x] 3.2 实现统一 `ToolRouter`：内置工具名与 MCP 工具名解析到唯一实现；未知工具名返回结构化错误且不执行
- [x] 3.3 将提供商/工具错误分类（超时、429、认证失败等），供编排层重试策略消费

## 4. MCP 集成

- [x] 4.1 实现 MCP 客户端：连接、发现工具列表、调用与超时/重试语义（`mcp-integration` spec）
- [x] 4.2 将 MCP 工具注册进 `ToolRouter`，并处理与内置工具的命名空间/前缀冲突（与设计中的前缀策略一致）

## 5. 内置工具与策略

- [x] 5.1 按 `builtin-tools` spec 分阶段实现核心工具（read、write、edit、grep、bash 等）的最小契约与 JSON schema
- [x] 5.2 实现调用前权限/策略层（高危操作需确认、工作区根、超时与输出上限），并补充审计或日志钩子（与设计一致）

## 6. 技能（Skills）

- [x] 6.1 实现技能发现与元数据解析，以及注入 prompt/工具列表的边界（`skills` spec）
- [x] 6.2 覆盖技能与工具/MCP 交互的契约测试

## 7. 插件扩展（跨平台优先）

- [x] 7.1 按 `design.md` §7 落地首版机制：编译期注册、静态链接或子进程 JSON-RPC 等，**避免**将纯 `plugin` 动态库作为唯一路径
- [x] 7.2 实现插件生命周期与缩小后的钩子表面（`plugin-extension` spec），并文档化平台与构建约束

## 8. Agent 运行时

- [x] 8.1 实现可配置的压缩/摘要策略及阈值（`agent-runtime` delta）
- [x] 8.2 实现可配置的重试策略及与持久化协同的回退/恢复路径（不静默丢数据）
- [x] 8.3 在启用结构化输出时校验 schema，校验失败不记为成功助手消息
- [x] 8.4 明确消息模型分阶段策略：读兼容/写兼容或一期字段对齐（与 Open Questions 决议一致），并在 API/存储中暴露版本或等价标识

## 9. HTTP 控制面

- [x] 9.1 实现会话列表端点（过滤规则以 spec 为准）与稳定 JSON 字段
- [x] 9.2 实现消息历史分页端点（cursor/limit），保证顺序与因果一致
- [x] 9.3 提供 OpenAPI 或机器可读契约端点，并与实现同步维护
- [x] 9.4 为新端点统一错误体与鉴权（与现有 `/v1` 基线一致）

## 10. ACP 桥接

- [x] 10.1 在现有 HTTP 服务上挂载 ACP 路由，复用同一鉴权（`acp-bridge` spec 与设计 §8）
- [x] 10.2 将 ACP 事件/会话映射到现有 `runtime` 会话模型，并补充集成测试

## 11. CLI 与配置

- [x] 11.1 扩展子命令集（会话/项目/工具/技能等，命名在实现中固定并写文档），未知子命令退出码 2
- [x] 11.2 提供交互式模式（TUI 或 REPL），复用与 `serve` 相同的配置加载规则
- [x] 11.3 新增仅 Go 使用的配置键置于 `x_opencode_go` 命名空间并文档化

## 12. 测试与文档收口

- [x] 12.1 为 HTTP、CLI、MCP、关键工具路径补充集成或契约测试（满足 `go-codebase-layout` delta 中「默认可在无网络环境运行」的约束，网络测试用 build tag 隔离）
- [x] 12.2 更新用户可见文档：新端点、配置键、插件/ACP 限制与上游差异说明
