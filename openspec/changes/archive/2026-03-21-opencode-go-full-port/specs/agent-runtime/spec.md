# 能力：agent-runtime

## ADDED Requirements

### Requirement: 会话生命周期

系统应当支持创建、选择与关闭**会话**（或等价工作单元），使每个会话在进程存活期内或显式关闭前具有稳定标识符。同进程内活跃会话的标识符必须唯一。

#### Scenario: 新会话获得标识符

- **当** 客户端通过支持的接口（CLI 或 HTTP API）请求新会话
- **则** 系统应当返回非空会话标识符，并在关闭前将该会话与后续操作关联

### Requirement: 上下文取消

包含模型轮次与工具调用的长时操作应当遵守 `context.Context` 取消：当上下文被取消时，系统应当停止为该会话当前轮次调度新工作，并在文档化的关闭时限内向调用方返回取消错误。

#### Scenario: 取消后停止工作

- **当** 调用方取消进行中的智能体操作所关联的上下文
- **则** 系统在观测到取消后不得再为该操作输出助手内容，并将取消传递给调用方

### Requirement: 消息顺序

在同一会话内，同一对话线程上的用户与助手消息在启用持久化时应当按因果顺序写入：对某一 turn，助手内容不得记录在其所回复的用户消息之前。

#### Scenario: 单轮有序持久化

- **当** 同一次 turn 内先接受用户消息再生成助手回复
- **则** 若有存储写入，应当先序列化该 turn 的用户消息，再序列化助手消息

### Requirement: 可观测进度

系统应当对主要生命周期事件输出机器可读或结构化日志信号（例如 slog 字段）：会话开始、轮次开始、轮次结束、工具调用开始、工具调用结束、致命错误。事件键名必须在同一小版本系列内保持稳定。

#### Scenario: 记录工具调用

- **当** 在某一 turn 内发起工具调用
- **则** 至少一条日志记录应当包含工具名与会话标识字段
