# 能力：http-api

## ADDED Requirements

### Requirement: 首版提供 HTTP 服务

首版系统应当暴露 HTTP API 服务器，实现远程或编辑器客户端所需的、`agent-runtime`、`llm-and-tools` 与 `persistence` 所要求的路由与行为。服务器应当可通过 `cli-and-config` 规定的 CLI 方式启动。

#### Scenario: 启动后开始监听

- **当** 用户使用有效配置启动 HTTP 服务器
- **则** 进程应当在配置的地址上打开 TCP 监听，并在优雅关闭前持续处理 HTTP 请求

### Requirement: API 版本前缀

所有 HTTP 路由应当归组在版本前缀下（例如 `/v1`）。破坏兼容的路径或载荷变更必须提升主版本前缀或置于新前缀后；此类变更必须在发行说明中标注 **BREAKING**。

#### Scenario: 路由位于版本前缀下

- **当** 客户端请求已文档化的 API 端点
- **则** 请求路径应当以该发行定义的版本前缀开头

### Requirement: 默认绑定 loopback

默认监听地址应当为 loopback（例如 `127.0.0.1`），并文档化默认端口。绑定所有网卡（`0.0.0.0` 或 `::`）必须显式配置，且必须按下文要求启用鉴权。

#### Scenario: 非 loopback 必须鉴权

- **当** 配置的绑定地址不是仅 loopback
- **则** 除非已在启动时配置并校验鉴权凭据，否则服务器不得启动

### Requirement: 鉴权基线

系统应当至少支持以下之一：静态 Bearer token 校验，或通过已配置头或查询参数进行的共享密钥校验（生产中非默认用法须文档说明）。鉴权配置必须可通过与 `cli-and-config` 相同的优先级规则加载。

#### Scenario: 需要鉴权时拒绝未认证请求

- **当** 当前绑定模式要求鉴权且请求缺少有效凭据
- **则** 服务器应当返回 HTTP 401，且不得执行特权操作

### Requirement: 优雅关闭

服务器应当在 SIGTERM 或平台等效的停止信号下优雅关闭：拒绝新连接，并在可配置的宽限期内完成进行中的请求后退出。

#### Scenario: 关闭时排空

- **当** 收到关闭信号
- **则** 监听器应当关闭，进行中的 HTTP handler 应当在宽限截止时间前得到完成机会

### Requirement: 错误响应

API 错误应当返回 JSON 体，且 `error` 对象形状稳定，包含机器可读的 `code` 字符串与人类可读的 `message` 字符串，除非请求显式协商非 JSON 响应。HTTP 状态码必须与语义一致（4xx 客户端，5xx 服务端）。

#### Scenario: 错误请求返回 JSON 错误

- **当** 客户端向 JSON 端点发送格式错误的 JSON 体
- **则** 响应应当为 4xx 状态，且响应体为符合文档 schema 的 JSON 错误

### Requirement: 流式响应选项

若实现通过 HTTP 暴露流式助手输出，应当使用已文档化的机制（例如分块传输或 SSE）。非流式客户端必须仍能通过已文档化的替代端点或查询参数获得完整响应。

#### Scenario: 流式行为有文档

- **当** 某补全端点启用流式
- **则** 该端点的 API 文档应当说明内容类型、分块格式与结束条件
